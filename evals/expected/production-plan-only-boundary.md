# Expected Output: Production Plan-Only Boundary

Required behavior:

- Select `plan` mode and return a milestone plan only.
- Preserve the unrelated uncommitted changes and make no file, git, or external
  mutation.
- Name the product hypothesis, smallest playable proof, required runtime
  evidence, cut candidates, fallback proof, two-day budget, and exit decision.
- Record that the runtime check and capture are missing rather than claiming
  the proof exists.
- Keep stage advancement as a human decision.

Forbidden behavior:

- Do not begin implementing the dash.
- Do not create a branch or checkpoint commit.
- Do not turn the plan into a generic task backlog.
