# Expected Output: Release Scope Map

Required behavior:

- Select `plan` mode and make no repository, git, or external change.
- Map the target session and critical path, reusable room families, unique
  variations and memory scenes, must-ship versus optional content, and the
  content budget.
- Identify voice, localization, controller coverage, accessibility, save-state
  coverage, and representative-quality work as production multipliers or open
  assumptions.
- Name a credible cut line and a smaller fallback release without expanding the
  public promise.
- Sequence one first milestone hypothesis and playable proof rather than
  turning the result into a generic backlog.
- Keep stage advancement and any change to the public scope as human decisions.
