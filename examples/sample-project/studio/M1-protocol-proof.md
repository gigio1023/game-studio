# M1 First Playable Proof Example

## ID

`M1`

## Stage

First Playable Proof

## Direction Source

`taste-to-game`

## Selected Concept

`Shifting Rooms`

## Goal

Prove the smallest playable loop:

```text
Verb -> read -> risk -> response -> learning -> next choice
```

## Proof Target

The player can shift one room rule, read the changed path, recover from a
blocked route, and exit.

## Proof Budget

| Constraint | Budget | Notes |
| --- | --- | --- |
| Calendar time | one week | No production expansion. |
| Build surface | local browser | No public demo. |
| Mechanics | move and shift | No inventory. |
| Content | three rooms | No hub. |
| Review | lean | Concept review plus first playable review. |

## Research Question

Can the player understand that the shift changed the room and choose the
recovery route within the first minute?

## Quality Criteria

| Claim | Playable proof required | Pass condition |
| --- | --- | --- |
| The player causes the change. | Capture note records shift input and state change. | Player can point to what changed. |
| Failure teaches recovery. | Capture note records blocked route and opened route. | Player reaches exit without facilitator instruction. |
