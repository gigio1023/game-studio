# Sample Project

This fictional example shows a small `taste-to-game` path through discovery,
concept selection, constraint setting, first prototype, first playable proof,
review, and next proof.

The capture artifacts in this sample are text notes that describe fictional
external captures. No binary video files are included or required.

## Profile

- engine: generic 2D browser runtime
- scope: solo creator, one week for first playable proof
- review mode: lean
- decision owner: human creator
- public promise status: none

## Discovery To Proof Flow

1. Taste profile records the creator's preferences without forcing genre terms.
2. Concept portfolio compares three small, rejectable concepts.
3. Creative brief selects one concept as candidate direction.
4. Constraint ladder turns the concept into a proof budget.
5. First prototype contract tests one risk before broad production.
6. First playable proof contract defines the executable build and capture note.
7. Reviews decide whether to advance, repeat, narrow, park, or reject.
8. M2 retest contract materializes the next proof after `CONCERNS`.
