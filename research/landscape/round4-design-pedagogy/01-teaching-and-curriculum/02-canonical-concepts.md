# Teaching And Curriculum Canon For Beginner Design Judgment: Canonical Concepts

[Back to index](../01-teaching-and-curriculum.md)

## Canonical Concepts

### Meaningful Play And Formal Vocabulary

*Rules of Play* is still the cleanest source for giving beginners words before
giving them judgments. A novice often says "it feels satisfying" or "it feels
like that game." The pedagogical move is to ask: what rule, action, feedback,
information, uncertainty, social relation, or cultural signal made the moment
meaningful?

Agent implication: do not convert beginner taste directly into genre or feature
lists. First translate taste into formal terms: rules, player action,
perceivable outcome, uncertainty, feedback, state change, social relation, and
fictional/cultural framing.

### Playcentric Iteration

*Game Design Workshop* and the MIT/NYU course structures converge on the same
teaching pattern: make something playable, test it, observe, revise, and
communicate what changed. The important curriculum point is that playtesting is
not a late quality check. It is how the designer learns what the design is.

Agent implication: every beginner-facing concept should leave with a prototype
question, a playtest scenario, and a revision rule. "This is a cool idea" is not
yet design judgment; "this is the behavior the next prototype must reveal" is.

### Lens-Based Judgment

Schell's lenses are pedagogically useful because they teach designers to ask
bounded questions from multiple perspectives. This avoids two novice failure
modes: one favorite mechanic dominating every decision, and one generic rubric
flattening the game's identity.

Agent implication: when reviewing a beginner concept, pick two to five relevant
lenses rather than applying the entire canon. A cozy gardening game does not
need the same first questions as a precision platformer. The agent should say
which lens it is using and why.

### Fun As Learning

Koster's enduring value is not a universal definition of fun; it is a diagnostic
question: what pattern is the player learning, and when does mastery turn into
boredom? This is especially useful for beginners who mistake content quantity
for depth.

Agent implication: ask "what does the player get better at?" before asking for
more enemies, rooms, cards, dialogue, or quests. If there is no changing player
skill, the design may need discovery, expression, narrative, social play, or
sensory craft rather than more progression.

### Feel As A Primary Verb Discipline

Swink makes "feel" teachable by breaking it out of intuition. For beginners,
this turns taste from "I want movement to feel good" into control response,
input timing, acceleration, camera, animation, sound, hit pause, recovery,
metaphor, and legibility.

Agent implication: every action game or movement-heavy concept needs a primary
verb lab before content expansion. The agent should preserve the intended feel
adjectives, then ask which input and feedback variables will prove them.

### Engineering Experiences

Sylvester is useful for translating emotion into design machinery without
pretending games are only machinery. His framing supports the idea that games
are engineered experiences: mechanics, fiction, pacing, rewards, social systems,
and production focus all shape emotional output.

Agent implication: beginners should be taught to name emotional triggers and
their delivery system. "Tense" can come from scarcity, time pressure, social
risk, fragile information, fiction, audio, camera, or failure cost. The agent
should propose alternatives and ask the creator to choose which one is theirs.

### Comparative Design Dimensions

*Characteristics of Games* is valuable because it turns taste into comparison:
luck versus skill, downtime, interaction, reward, effort, player count,
complexity, and balance. Beginners need these dimensions because they can
compare games before they can invent systems confidently.

Agent implication: after a taste interview, ask structured comparison questions:
"Do you like this reference because it reduces downtime, because luck softens
failure, because skill expression is visible, or because social interaction
creates tension?" This builds judgment without demanding jargon first.

### MDA And Its Limits

MDA remains the most useful beginner chain: mechanics create dynamics, dynamics
create aesthetic responses, and players perceive that chain in reverse. It
teaches causal thinking. But DDE, Sicart, and Lantz explain why MDA cannot be
treated as the whole truth: assets, fiction, UI, audio, embodiment, player
interpretation, culture, and aesthetic thought are not merely downstream from
mechanics.

Agent implication: use MDA as a scaffold, not a verdict. If a concept is mostly
about movement, economy, or challenge, MDA is a strong first pass. If it is
about narrative, ethics, beauty, horror, discomfort, personal expression, or
player appropriation, include DDE/Sicart/Lantz-style questions about experience,
materials, meaning, and play.

### DIY Authorship And Personal Voice

Anthropy's value for this repo is not only democratization. It is a guardrail:
beginners should not be optimized into generic competent taste. Personal
background, odd references, rough edges, and self-expression can be legitimate
design material.

Agent implication: agent guidance should distinguish "unclear" from "unmarketed"
or "unconventional." The agent may ask for structure, proof, and scope, but it
should not normalize away the user's strange or personal center.

### Play As Appropriation

Sicart's critique matters for pedagogy because beginners often think the
designer controls meaning. Players misunderstand, exploit, roleplay, resist,
and create meanings the designer did not intend. Lantz's aesthetics of games
extends this: games create beauty and meaning through thought, form, and play,
not only through rule output.

Agent implication: playtest notes should include surprising appropriation, not
only failure against intent. If players discover a better play pattern, the
agent should present it as a design fork: preserve, support, constrain, or cut.

### Level Design As Teaching Through Space

Level design pedagogy is especially relevant because it turns abstract design
judgment into visible sequence. Totten, Valve's level-flow notes, NYU's level
design studio, and level-design playtesting guidance all point to the same
teaching move: a level is a curriculum. It introduces affordances, creates
expectations, tests understanding, varies pressure, and gives recovery space.

Agent implication: for spatial games, the agent should ask what the level
teaches before what it contains. The first room, first obstacle, first vista,
first fork, first reward, and first failure are instructional design, not just
content.

### Playtesting As Pedagogy

Fullerton, MIT, NYU, The Level Design Book, NN/g, and games user research all
support the same core practice: observe realistic users doing realistic tasks,
then separate observation from interpretation. Beginners need this distinction
because they often overreact to opinions and underreact to behavior.

Agent implication: playtest summaries should separate: research question,
participant fit, task/scenario, observed behavior, player quote, designer
inference, severity/confidence, proposed change, and retest condition.
