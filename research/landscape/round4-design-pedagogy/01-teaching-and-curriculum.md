# Teaching And Curriculum Canon For Beginner Design Judgment

This round focuses on game design pedagogy: how a beginner moves from taste,
reference moments, and vague preference into structured design judgment without
losing creative authorship. The repo already cites much of the canon. The
remaining gap is less "what should agents cite?" and more "how should agents
teach the designer to see?"

## Canonical Concepts

### Meaningful Play And Formal Vocabulary

*Rules of Play* is still the cleanest source for giving beginners words before
giving them judgments. A novice often says "it feels satisfying" or "it feels
like that game." The pedagogical move is to ask: what rule, action, feedback,
information, uncertainty, social relation, or cultural signal made the moment
meaningful?

Agent implication: do not convert beginner taste directly into genre or feature
lists. First translate taste into formal terms: rules, player action,
perceivable outcome, uncertainty, feedback, state change, social relation, and
fictional/cultural framing.

### Playcentric Iteration

*Game Design Workshop* and the MIT/NYU course structures converge on the same
teaching pattern: make something playable, test it, observe, revise, and
communicate what changed. The important curriculum point is that playtesting is
not a late quality check. It is how the designer learns what the design is.

Agent implication: every beginner-facing concept should leave with a prototype
question, a playtest scenario, and a revision rule. "This is a cool idea" is not
yet design judgment; "this is the behavior the next prototype must reveal" is.

### Lens-Based Judgment

Schell's lenses are pedagogically useful because they teach designers to ask
bounded questions from multiple perspectives. This avoids two novice failure
modes: one favorite mechanic dominating every decision, and one generic rubric
flattening the game's identity.

Agent implication: when reviewing a beginner concept, pick two to five relevant
lenses rather than applying the entire canon. A cozy gardening game does not
need the same first questions as a precision platformer. The agent should say
which lens it is using and why.

### Fun As Learning

Koster's enduring value is not a universal definition of fun; it is a diagnostic
question: what pattern is the player learning, and when does mastery turn into
boredom? This is especially useful for beginners who mistake content quantity
for depth.

Agent implication: ask "what does the player get better at?" before asking for
more enemies, rooms, cards, dialogue, or quests. If there is no changing player
skill, the design may need discovery, expression, narrative, social play, or
sensory craft rather than more progression.

### Feel As A Primary Verb Discipline

Swink makes "feel" teachable by breaking it out of intuition. For beginners,
this turns taste from "I want movement to feel good" into control response,
input timing, acceleration, camera, animation, sound, hit pause, recovery,
metaphor, and legibility.

Agent implication: every action game or movement-heavy concept needs a primary
verb lab before content expansion. The agent should preserve the intended feel
adjectives, then ask which input and feedback variables will prove them.

### Engineering Experiences

Sylvester is useful for translating emotion into design machinery without
pretending games are only machinery. His framing supports the idea that games
are engineered experiences: mechanics, fiction, pacing, rewards, social systems,
and production focus all shape emotional output.

Agent implication: beginners should be taught to name emotional triggers and
their delivery system. "Tense" can come from scarcity, time pressure, social
risk, fragile information, fiction, audio, camera, or failure cost. The agent
should propose alternatives and ask the creator to choose which one is theirs.

### Comparative Design Dimensions

*Characteristics of Games* is valuable because it turns taste into comparison:
luck versus skill, downtime, interaction, reward, effort, player count,
complexity, and balance. Beginners need these dimensions because they can
compare games before they can invent systems confidently.

Agent implication: after a taste interview, ask structured comparison questions:
"Do you like this reference because it reduces downtime, because luck softens
failure, because skill expression is visible, or because social interaction
creates tension?" This builds judgment without demanding jargon first.

### MDA And Its Limits

MDA remains the most useful beginner chain: mechanics create dynamics, dynamics
create aesthetic responses, and players perceive that chain in reverse. It
teaches causal thinking. But DDE, Sicart, and Lantz explain why MDA cannot be
treated as the whole truth: assets, fiction, UI, audio, embodiment, player
interpretation, culture, and aesthetic thought are not merely downstream from
mechanics.

Agent implication: use MDA as a scaffold, not a verdict. If a concept is mostly
about movement, economy, or challenge, MDA is a strong first pass. If it is
about narrative, ethics, beauty, horror, discomfort, personal expression, or
player appropriation, include DDE/Sicart/Lantz-style questions about experience,
materials, meaning, and play.

### DIY Authorship And Personal Voice

Anthropy's value for this repo is not only democratization. It is a guardrail:
beginners should not be optimized into generic competent taste. Personal
background, odd references, rough edges, and self-expression can be legitimate
design material.

Agent implication: agent guidance should distinguish "unclear" from "unmarketed"
or "unconventional." The agent may ask for structure, proof, and scope, but it
should not normalize away the user's strange or personal center.

### Play As Appropriation

Sicart's critique matters for pedagogy because beginners often think the
designer controls meaning. Players misunderstand, exploit, roleplay, resist,
and create meanings the designer did not intend. Lantz's aesthetics of games
extends this: games create beauty and meaning through thought, form, and play,
not only through rule output.

Agent implication: playtest notes should include surprising appropriation, not
only failure against intent. If players discover a better play pattern, the
agent should present it as a design fork: preserve, support, constrain, or cut.

### Level Design As Teaching Through Space

Level design pedagogy is especially relevant because it turns abstract design
judgment into visible sequence. Totten, Valve's level-flow notes, NYU's level
design studio, and level-design playtesting guidance all point to the same
teaching move: a level is a curriculum. It introduces affordances, creates
expectations, tests understanding, varies pressure, and gives recovery space.

Agent implication: for spatial games, the agent should ask what the level
teaches before what it contains. The first room, first obstacle, first vista,
first fork, first reward, and first failure are instructional design, not just
content.

### Playtesting As Pedagogy

Fullerton, MIT, NYU, The Level Design Book, NN/g, and games user research all
support the same core practice: observe realistic users doing realistic tasks,
then separate observation from interpretation. Beginners need this distinction
because they often overreact to opinions and underreact to behavior.

Agent implication: playtest summaries should separate: research question,
participant fit, task/scenario, observed behavior, player quote, designer
inference, severity/confidence, proposed change, and retest condition.

## Translation Into Agent Guidance

### 1. Teach Taste As Evidence, Not As A Command

Beginner taste is valid input, but not yet instruction. Agents should treat
references as evidence about desired play, not as features to clone.

Use this sequence:

1. Preserve the user's words.
2. Ask what moment, not what whole game.
3. Extract the player action.
4. Extract the perceived consequence.
5. Extract the emotional or aesthetic response.
6. Offer two or three possible design interpretations.
7. Ask the user to choose, combine, or reject.

This keeps the author in control while raising the quality of the design
conversation.

### 2. Convert Preference Into Design Dimensions

When a user says "I like this," agents should not jump to "then make a roguelike"
or "then add crafting." Use comparative dimensions first:

- luck or skill
- planning or improvisation
- mastery or discovery
- expression or optimization
- social pressure or solitary focus
- punishment or recovery
- scarcity or abundance
- clarity or mystery
- authored sequence or open-ended play
- short-session satisfaction or long-arc investment

These dimensions let beginners build judgment by comparing familiar experiences.

### 3. Use MDA Backward, Then Reopen The Frame

For each proposed concept, require a small causal chain:

```text
Desired player experience:
Runtime dynamic that could create it:
Mechanic or content rule that could produce that dynamic:
What the player must understand:
What a playtest should show:
```

Then reopen the frame with DDE/Sicart/Lantz questions:

```text
Which non-mechanical materials matter here: visual, audio, UI, narrative, space?
What might players appropriate or misunderstand?
What beauty, meaning, or thought does the game ask players to inhabit?
What part of the user's personal taste must not be normalized away?
```

### 4. Make The Agent A Critic, Not A Co-Author By Default

The agent should not silently replace the beginner's taste with its own design.
Its first job is to make the user's taste more legible. Its second job is to
visible consequences. Its third job is to propose testable next moves.

Recommended response shape:

```text
What I think you are preserving:
What design problem that creates:
Two plausible ways to solve it:
What each option risks:
What prototype would reveal the truth:
Decision for you:
```

This avoids both extremes: unstructured encouragement and overbearing expert
correction.

### 5. Teach Through Small Assignments

Textbook-grade pedagogy is assignment-driven. Beginner-facing agents should
produce short, playable exercises:

- make a one-rule non-digital version of the core tension
- tune one number and predict the player behavior change
- remove one feature and observe whether the fantasy survives
- make one room that teaches the primary verb without text
- run a three-person blind playtest with one question
- write two rejected variants and why they are not the selected game

Each exercise should produce a decision artifact, not just a learning moment.

### 6. Protect Creative Roughness

Beginners often have taste before craft. The agent should distinguish:

- productive ambiguity: worth preserving until a prototype tests it
- expressive roughness: part of the author's voice
- missing decision: needs a choice
- missing craft: needs a technique
- incoherence: current elements contradict each other
- overreach: good intent but too much scope for the proof

Only the last three should be treated as correction targets.

## Missing Curriculum Modules

### Module 1: Taste Literacy And Reference Decomposition

Purpose: turn "I like X" into structured observations without flattening voice.

Core sources: *Rules of Play*, *A Game Design Vocabulary*, MDA, *The Art of Game
Design*, Anthropy.

Exercises:

- reference moment sheet
- verbs/objects/resistance extraction
- "what not to copy" list
- two alternate interpretations of the same taste statement
- creator choice record

Artifact: `reference-constraint-sheet.md`

### Module 2: Formal Elements Lab

Purpose: teach rules, goals, actions, resources, conflict, boundaries,
uncertainty, feedback, and outcome as manipulable materials.

Core sources: *Rules of Play*, *Game Design Workshop*, *Characteristics of
Games*, *A Game Design Vocabulary*.

Exercises:

- redesign a familiar physical game by changing one formal element
- compare two reference games by luck/skill/effort/downtime
- identify the first meaningful choice and its feedback

Artifact: `formal-elements-map.md`

### Module 3: Experience Chain Workshop

Purpose: teach causal design reasoning without treating MDA as a universal
truth.

Core sources: MDA, DDE, Sicart, Lantz, *Designing Games*.

Exercises:

- write mechanic -> dynamic -> experience chains
- reverse a reference moment from experience back to mechanics
- add non-mechanical materials: sound, camera, UI, fiction, space
- identify one likely player appropriation

Artifact: `experience-chain.md`

### Module 4: Primary Verb And Feel Lab

Purpose: convert feel adjectives into input, response, camera, animation,
audio, and recovery variables.

Core sources: *Game Feel*, *A Theory of Fun*, *The Gamer's Brain* if retained
from existing repo sources.

Exercises:

- one-button prototype
- response-time comparison
- failure recovery test
- "fun after content removed" check

Artifact: `primary-verb-feel-sheet.md`

### Module 5: Level As Lesson

Purpose: teach spatial design as player education.

Core sources: Totten, Valve Level Flow, NYU Level Design Studio, The Level Design
Book, level design patterns literature.

Exercises:

- first-room teaching sketch
- landmark/fork/reward/recovery map
- path-flow critique
- blind navigation playtest

Artifact: `level-teaching-plan.md`

### Module 6: Playtest Facilitation And Interpretation

Purpose: teach beginners how to learn from players without obeying every
opinion.

Core sources: *Game Design Workshop*, MIT/NYU studio pedagogy, The Level Design
Book, NN/g usability testing, *Games User Research*.

Exercises:

- write one research question
- recruit participant fit
- facilitate without explaining
- separate observation, quote, inference, and change
- define retest condition

Artifact: `beginner-playtest-report.md`

### Module 7: Critique Without Authorial Collapse

Purpose: teach agents and human reviewers how to improve clarity while
preserving personal voice.

Core sources: Anthropy, Sicart, Lantz, Schell, NYU critique/collaboration model.

Exercises:

- name what must be preserved
- name what is unclear
- identify the design cost of preserving it
- propose two fixes with different authorial tradeoffs
- user selects the direction

Artifact: `taste-preservation-decision.md`

### Module 8: Design Judgment Portfolio

Purpose: move the beginner from isolated exercises to reusable judgment.

Core sources: MIT/NYU project-based studio structure, *Game Design Workshop*,
Schell, Fullerton, *Characteristics of Games*.

Exercises:

- three small playable studies
- one postmortem per study
- one rejected concept with reusable learnings
- one selected first playable proof

Artifact: `beginner-design-portfolio.md`

## Repo Implications

### What The Repo Already Has

- A strong craft reference index in `core/references/game-craft-reference-index.md`.
- A beginner entry route in `docs/beginner-taste-to-game.md`.
- Taste intake fields in `core/templates/intake/beginner-taste-interview.md`.
- Good recognition that playtest evidence matters in
  `research/foundations/qa-playtesting-and-accessibility.md`.
- A useful next-step insight in
  `docs/improvement/value-review/13-craft-reference-curriculum.md`: syllabus is
  not curriculum, and operational guides need worked examples.
- Prior research already names MDA's limits and the need for DDE/Sicart/Lantz
  critique in `research/landscape/synthesis.md`.

### What Is Still Missing

- The repo has canonical citations, but not enough beginner exercises that
  convert taste into practiced judgment.
- The taste interview captures references, feelings, and boundaries, but it
  does not yet force reference decomposition into verbs, objects, resistance,
  feedback, and "what not to copy."
- The existing craft reference index lists *Game Design Workshop*, *Rules of Play*, MDA,
  *Characteristics of Games*, and *Game Feel*, but it does not encode the
  classroom sequence: non-digital exercise, prototype, critique, playtest,
  revise, postmortem.
- The repo has review rubrics, but beginner pedagogy needs assignment templates
  that produce evidence before formal review.
- MDA is present, and DDE critique is present in research, but beginner-facing
  prompts need a safe way to use MDA without turning the eight aesthetics into a
  restrictive taste menu.
- Anthropy/Sicart/Lantz are present as philosophical references, but their
  operational role should be clearer: protect authorial voice, player
  appropriation, and aesthetic meaning from overly procedural review.
- Level design appears as a review workflow, but not enough as a teaching
  module: level as lesson, path as argument, first room as tutorial, flow as
  player need.
- Playtesting guidance exists, but beginner-facing templates should more
  aggressively separate observation from interpretation and player preference
  from behavioral evidence.

### Core Thesis

The repo should treat beginner design education as a ladder:

```text
taste -> moment -> action -> feedback -> formal element -> dynamic ->
experience -> prototype question -> observed player behavior -> revision
```

At every step, the agent should preserve a visible author decision. The agent
may structure, compare, question, and propose, but the user should remain the
source of taste and final selection.
