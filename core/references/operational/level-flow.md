# Level Flow Operational Guide

Use this guide when a room, route, puzzle, encounter, hub, tutorial, or arena
claims to teach, pace, or direct player movement.

## Source Claim

Level flow is the player's readable path through attention, choice, action,
feedback, and recovery. A level review should check what the player sees and
learns before checking how much content exists.

## Operating Moves

### 1. Name the first read

State what the player should notice first and why. Use landmarks, motion,
lighting, contrast, framing, sound, or enemy behavior as reasons.

Worked example:

```text
First read: the open blue doorway on the north wall.
Reason: it is the only moving light source and sits behind the first safe gap.
Risk: the background window uses the same blue and may compete for attention.
```

### 2. Sequence teach, practice, vary, combine

Do not test a rule before the level has taught it under low pressure.

Worked example:

```text
Teach: one safe room shift opens a visible door.
Practice: the next room shift closes one route and opens another.
Vary: a blocked route forces observation before shifting.
Combine: final room asks the player to shift after noticing both exits.
```

### 3. Add a recovery path for each learning failure

If the player misunderstands the rule, the level should provide a readable way
back into learning instead of only punishment.

Worked example:

```text
Failure: player shifts too early and blocks the east route.
Recovery: north route opens in view; the blocked door stays visible; the player
can return to the switch and compare states.
```

### 4. Check pacing by player attention, not room count

One room can be too much if it introduces multiple rules at once. Three rooms
can be fine if each room tests one clear idea.

Worked failure:

```text
The level introduces a switch, enemy patrol, locked door, collectible, and
camera pan in the first room. The player cannot tell which element matters.
Review decision: cut to one rule until the first read and recovery path are
clear.
```

## Decision Template

```text
Section or room:
First read:
Teaching beat:
Practice beat:
Variation or combination:
Failure and recovery:
Attention conflict:
Capture or walkthrough checked:
Level-flow verdict:
Next build or retest:
```

## Failure Modes This Guide Catches

- The level tests before it teaches.
- Decoration competes with the critical path.
- Recovery is too far from the mistake to support learning.
- The route requires an explanation outside play.
- The first room introduces more rules than the player can hold.

## Anti-Pattern

Do not approve level flow because the layout is complete. Approve it when the
player can notice the next useful thing, act on it, understand the result, and
recover after a mistake.
