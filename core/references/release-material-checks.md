# Release Material Checks

Use this when a project prepares a public demo, store page, trailer, press kit,
README, website, or release candidate.

## Public Claim Map

Every public claim should map to:

- build identifier
- capture, screenshot, or replay
- known limitation
- owner
- review verdict
- update trigger

## Required Release Materials

- store or download page
- screenshots and trailer
- controls and input support
- accessibility feature disclosure
- localization scope
- credits and third-party licenses
- support contact or issue route
- known limitations
- save data and compatibility notes when relevant

## Blockers

- A public claim has no build truth.
- A screenshot or trailer shows cut or unavailable behavior.
- Accessibility or localization language is broader than tested support.
- Asset license or credit ownership is unknown.
- Known blocking bugs are hidden instead of disclosed, fixed, or waived.
