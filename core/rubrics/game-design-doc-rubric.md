# Game Design Document Rubric

This rubric checks whether a design document can guide production.

## Must contain

- Player role
- Core fantasy
- Core loop
- Success and failure states
- Inputs and outputs
- Progression or session structure
- Feedback surfaces
- Edge cases
- Tuning knobs
- Acceptance criteria

## Must avoid

- Generic genre claims without playable consequence.
- Lore that does not affect play.
- Systems with no owner.
- AI behavior with no deterministic boundary.
- "Polish later" as a substitute for an experience target.

## Verdicts

- `READY`: Usable for implementation.
- `CONCERNS`: Usable if listed gaps are accepted.
- `NOT_READY`: Too ambiguous for implementation.
