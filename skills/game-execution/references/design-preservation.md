# Design Preservation

Use this before claiming implementation work is complete.

## Check

- Which player action changed?
- Which feedback changed?
- Which player-facing risk was introduced?
- Which direction pillar did the implementation preserve?
- Which alternative was rejected?
- Which manual review is still needed?
- Which playable proof shows the intended behavior?
- Which internal evidence records preserve that proof?

## Handoff Rule

When the task touches gameplay, UI, camera, audio, narrative, progression, or
release copy, include a short design-preservation note in the handoff.
