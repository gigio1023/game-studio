# Design Foundations Review

Use this before a director verdict when the work affects game identity or player experience.

## Lens Set

- **Player role**: Who is the player in the system and in the fantasy?
- **Core action**: What does the player do repeatedly?
- **Meaningful choice**: What changes because the player chose one action over another?
- **Feedback**: How does the game show success, failure, danger, and progress?
- **Learning**: What pattern does the player learn first?
- **MDA chain**: Which mechanic creates which dynamic and intended experience?
- **Anti-pillar**: Which tempting feature or tone should be rejected?
- **Proof**: What playable proof shows this is happening?

## Operational Guides

- For the MDA chain, use `.game-studio/core/references/operational/mda.md`.
- For first-session spatial learning or teaching order, use

## Reject When

- The answer is only a story premise.
- The answer is only a feature list.
- The answer uses "fun" without naming observed player behavior.
- The answer cannot identify the first proof.
- The answer treats AI preference as the creator's taste or Direction Lock.
