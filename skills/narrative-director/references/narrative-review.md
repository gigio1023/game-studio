# Narrative Review

Narrative review checks whether story works as playable design.

Use `.game-studio/core/references/operational/storylets.md` when the work
contains branching beats, repeatable narrative units, AI-authored text, or
state-dependent story delivery.

## Flow

1. State the story promise in one sentence.
2. Name the player role in the story.
3. List critical beats and player verbs.
4. Check state changes, branch cost, and content budget.
5. Check localization, accessibility, and content QA risk.
6. Check AI authority if generated text is involved.
7. Return a verdict.

## Output

```text
Verdict:
Story promise:
Playable beats:
State consequences:
Content budget:
Localization or accessibility risk:
AI or canon boundary:
Required next proof:
```
