# Player Walkthrough Review

Use this when the project needs a player-centered pass before or after a
playable proof. Before a build exists, treat this as weak proof.

## Method

1. Select one target player and session context.
2. Use the taste profile or selected audience when discovery created the
   concept.
3. Walk through the first visual impression, first input, first success, first
   failure, recovery, stopping point, and reason to return.
4. State only what the current design or build proves.
5. Mark unspecified moments as blind spots.

## Checkpoints

- first visual impression
- first meaningful input
- first "I understand this" moment
- first failure and recovery
- first meaningful choice
- first natural stopping point
- first reason to return
- first public-promise touch, if relevant
