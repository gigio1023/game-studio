# Accessibility Review

Use this when player-facing input, UI, text, audio, motion, difficulty,
communication, or public accessibility claims are in scope.

## Barrier Categories

- Motor: remapping, alternative input, hold/tap options, timing pressure.
- Vision: contrast, text size, color independence, object clarity, motion.
- Hearing: captions, subtitles, visual warnings, separate volume controls.
- Cognitive: clear goals, memory load, recovery, difficulty support.
- Speech: voice alternatives and communication settings.
- General: support expectations and known limitations.

## Questions

- Which core path could block a player?
- Is the barrier solved, bounded, disclosed, or out of scope?
- Which public claim depends on this support?
- What capture or review note preserves the finding?
- What retest is required after a fix?
