# UX Review

Use this when interface, onboarding, controls, state visibility, menus, text,
feedback, settings, or recovery paths affect the player experience.

## Questions

- What is the player's immediate goal?
- Which state must be visible?
- What input mapping is required?
- What error can the player make, and how do they recover?
- What text can be removed because the interface teaches it?
- Which accessibility path is affected?
- What proof shows the player understood the UI?

## Blocking Concerns

- The player cannot tell what to do next.
- The UI hides state, cost, cooldown, target, or consequence.
- Tutorial text compensates for unclear interaction.
- Settings or remapping are promised but not implemented.
- Recovery from an invalid action is unclear.
