# Milestone Planning

Milestones prove product hypotheses.

## Flow

1. Name the current stage.
2. State the product hypothesis.
3. Define non-goals.
4. List required evidence.
5. Assign gate reviewers.
6. Define the exit decision.

## Output

```text
Current stage:
Recommended milestone:
Product hypothesis:
Required artifacts:
Required evidence:
Gate reviewers:
Risks:
```

## Stage Rule

Move in this order unless the project already has evidence for the skipped stage:

1. Direction Lock
2. Protocol Proof
3. Core Loop Prototype
4. Presentation Value Gate
5. Vertical Slice
6. Demo Candidate
7. Release Candidate
