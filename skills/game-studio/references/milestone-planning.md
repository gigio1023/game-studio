# Milestone Planning

Milestones prove product hypotheses.

## Flow

1. Name the current stage.
2. State the product hypothesis.
3. Define non-goals.
4. State the smallest useful proof.
5. List required proof artifacts.
6. Name cut candidates and fallback proof.
7. Assign gate reviewers.
8. Define the exit decision.
9. Name internal evidence records needed to preserve the proof trail.


## Stage Rule

Move in this order unless the project already has proof for the skipped stage:

0. Creative Discovery
1. Entry Route
2. Taste Clarity or Direction Carry-In Audit
3. Concept Slate
4. Creative Brief
5. First Prototype Question
6. First Playable Proof
7. Direction Lock
8. Protocol Proof
9. Core Loop Prototype
10. Pre-production Exit
11. Presentation Value Gate
12. Vertical Slice
13. Demo Candidate
14. Release Candidate
