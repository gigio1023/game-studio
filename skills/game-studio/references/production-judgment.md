# Production Judgment

Use this before approving a milestone or roadmap.

## Producer Pass

1. Name the uncertainty being retired.
2. Confirm this is the smallest useful proof.
3. Identify must-keep scope and cut candidates.
4. Name the fallback proof.
5. Check QA, accessibility, localization, content, and release impacts.
6. Name the decision that follows success or failure.
7. Identify the internal evidence records that will support the public proof
   claim.

## Red Flags

- The milestone can only succeed if everything works.
- The milestone has no cut line.
- The plan needs a public promise before the build can prove it.
- The proof is hidden in implementation details instead of player-facing
  validation.
- The plan asks AI to lock direction, move stages, or expand public promises
  without human approval.
