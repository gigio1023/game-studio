# Substantive Review

Game Studio treats scripts as repository hygiene and Codex as the reviewer of game substance.

## Principle

Automated checks are useful, but they are not game direction. A project advances
when the right roles can defend the playable evidence.

## Codex-Led Review Surfaces

- `core/references/game-craft-source-map.md`: source-backed craft vocabulary.
- `core/references/codex-review-practice.md`: review method and verdict shape.
- `core/roles/playbooks/`: discipline-specific review behavior.
- `core/rubrics/`: readiness criteria for direction, production, playtest, accessibility, and moment-to-moment quality.
- `core/templates/`: project artifacts that force evidence, tradeoffs, and follow-up decisions.
- `research/`: source reports used to improve this framework.

## What To Avoid

- Do not treat a filled template as a good plan.
- Do not treat passing commands as playable quality.
- Do not let role names replace role judgment.
- Do not approve public claims without build evidence.
- Do not add more process when a sharper review question would solve the problem.

## Minimal Script Policy

Use scripts only for cheap invariants. For game quality, require a written role
review that names evidence, concerns, tradeoffs, and the next proof.
