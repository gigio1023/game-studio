# Plain Language Game Design Glossary

This glossary helps agents explain game-production terms without making novice
creators learn studio jargon before they can make decisions.

## Core Terms

### Player Fantasy

What the player imagines they are doing or becoming.

### Verb

The action the player performs repeatedly, such as move, talk, dodge, combine,
build, hide, trade, aim, or choose.

### Loop

The repeatable pattern of action, result, feedback, and next decision.

### Pillar

A rule for what the game must keep true when scope changes.

### Anti-Pillar

A rule for what the game must avoid becoming.

### Proof

A concrete playable or inspectable result that answers a design, production, or
release question.

### Playable Proof

A proof the player can experience directly, even if it uses placeholder art,
small content, or simulated systems.

### Proof Budget

The smallest amount of time, scope, content, and fidelity needed to answer the
current question.

### Internal Evidence

Records that support a proof, such as builds, captures, playtest notes, QA
results, decision records, and release audits.

### Direction Lock

The point where the game identity is stable enough for production decisions.
The design can still change, but the team should not silently change what game
is being made.

### Protocol Proof

The smallest proof that the central rule or interaction can work.

### Core Loop Prototype

A prototype where the player can repeat the main loop with feedback, failure,
and a reason to continue.

### Vertical Slice

A short section at intended shipped quality. It should not be used as a label
for an early prototype.

### Gate Verdict

A decision that says whether the project is `READY`, has `CONCERNS`, or is
`NOT_READY` for the requested move.

## Terms To Avoid Misusing

### Fun

Do not use as proof by itself. Name what players did, understood, repeated,
avoided, or asked to do next.

### Polished

Do not use as a replacement for clarity, feedback, performance, accessibility,
or release readiness.

### Content Complete

Do not use unless required content is listed and tied to release promises.

### AI-Generated

Do not treat generation as authority. Generated material still needs direction,
review, and proof.
