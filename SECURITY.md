# Security

Game Studio is a documentation and agent-skill framework. Security issues are
most likely to involve unsafe instructions, supply-chain assumptions, private
data exposure, or generated workflow guidance.

## Report A Security Issue

Do not open a public issue with exploit details, secrets, tokens, private paths,
or confidential project data.

Use GitHub private vulnerability reporting if it is enabled for the repository.
If that is unavailable, open a minimal public issue asking for a private contact
channel and omit sensitive details.

## What To Report

- Instructions that could expose secrets or private project data.
- Guidance that encourages unsafe dependency installation or command execution.
- Templates that ask contributors to publish confidential information.
- Security-sensitive workflow errors in adapter snippets or agent skills.

## What Not To Report Here

- General game-design disagreement.
- Broken links or documentation typos.
- Feature requests without a security impact.

Use the issue templates for non-security reports.
